# Reviewing Content

**Objetivo:** Validar qualidade, consistência e alinhamento com briefing de todas as entregas do projeto.

## Quando Usar

- Após cada tarefa (validação rápida)
- Após cada fase (validação de consistência)
- Após todas as tarefas (revisão final)
- Quando o usuário pede "revise", "valide", "check"

## Filosofia

**Revise cedo, revise frequentemente.**

Problemas detectados cedo são baratos de corrigir. Problemas detectados tarde são caros ou impossíveis.

## Tipos de Revisão

### 1. Revisão por Tarefa (Rápida)

Após cada entrega de subagent:

**Checklist:**
- [ ] Entregável existe no caminho correto?
- [ ] Critério de sucesso foi atendido?
- [ ] Brand voice está consistente?
- [ ] Não há erros óbvios (ortografia, formatação)?
- [ ] Dependências foram respeitadas?

**Tempo:** 1-2 minutos por tarefa

**Resultado:**
- ✅ Aprovado → marque como completa no ledger
- ⚠️ Aprovado com observações → note no ledger, prossiga
- ❌ Reprovado → forneça feedback, peça revisão

### 2. Revisão por Fase (Consistência)

Após completar uma fase:

**Checklist:**
- [ ] Todas as tarefas da fase estão completas?
- [ ] Entregas são consistentes entre si?
- [ ] Brand voice é uniforme em todas as peças?
- [ ] Mensagens estão alinhadas?
- [ ] Não há contradições entre entregas?

**Tempo:** 5-10 minutos por fase

**Resultado:**
- ✅ Fase aprovada → prossiga para próxima fase
- ⚠️ Fase aprovada com ajustes → note issues, corrija na próxima fase
- ❌ Fase reprovada → identifique problemas, peça revisões

### 3. Revisão Final (Completa)

Após todas as tarefas:

**Checklist Geral:**

#### Qualidade Individual
- [ ] Cada entrega atende seu critério de sucesso?
- [ ] Não há erros de ortografia/gramática?
- [ ] Formatação está profissional?
- [ ] Links estão funcionais?
- [ ] Imagens têm alt text?

#### Consistência Global
- [ ] Brand voice é uniforme em todas as peças?
- [ ] Tom é consistente (formal/informal, técnico/acessível)?
- [ ] Vocabulário segue brand-voice.md?
- [ ] Não há contradições entre peças?
- [ ] Mensagens-chave aparecem consistentemente?

#### Alinhamento com Briefing
- [ ] Objetivo do projeto foi atendido?
- [ ] Público-alvo foi corretamente endereçado?
- [ ] Canais escolhidos foram usados?
- [ ] Escopo foi respeitado (nem mais, nem menos)?
- [ ] Métricas de sucesso foram definidas?

#### SEO e Acessibilidade
- [ ] Artigos têm keyword primária e secundárias?
- [ ] Meta descriptions estão otimizadas?
- [ ] Headlines são benefit-driven?
- [ ] Imagens têm alt text descritivo?
- [ ] Links internos e externos estão presentes?

#### CTAs e Conversão
- [ ] Cada peça tem CTA claro?
- [ ] CTAs são adequados ao estágio do funil?
- [ ] CTAs são consistentes entre peças?
- [ ] Próximo passo é óbvio para o leitor?

#### Visuais
- [ ] Imagens seguem paleta de cores?
- [ ] Dimensões estão corretas para cada canal?
- [ ] Estilo é consistente?
- [ ] Qualidade é alta (sem pixelização, ruído)?

#### Documentos
- [ ] Formatação está profissional?
- [ ] Metadados estão completos (autor, data)?
- [ ] Numeração de páginas está presente?
- [ ] Sumário está presente (para documentos longos)?

**Tempo:** 15-30 minutos para projeto completo

**Resultado:**
- ✅ Projeto aprovado → prossiga para finalização
- ⚠️ Projeto aprovado com ajustes menores → note, corrija se rápido
- ❌ Projeto reprovado → identifique problemas críticos, peça revisões

## Técnicas de Revisão

### Leitura em Voz Alta

Para copy e textos curtos:
- Leia em voz alta para detectar frases estranhas
- Identifique ritmo e fluidez
- Detecte repetições e clichês

### Checklist Invertido

Para validar completude:
- Liste o que DEVERIA estar presente
- Marque cada item
- Identifique o que está faltando

### Comparação com Referências

Para validar qualidade:
- Compare com exemplos de referência (concorrentes, cases)
- Identifique gaps
- Sugira melhorias

### Revisão Cruzada

Para validar consistência:
- Leia peça A
- Leia peça B
- Verifique se tom, mensagem e estilo são consistentes
- Identifique contradições

## Reporte de Revisão

### Para Tarefa Individual

```markdown
## Revisão: Tarefa 2.1 — Artigo de Blog

**Status:** ✅ Aprovado

**Entregável:** `output/artigos/artigo-principal.md`

**Validação:**
- ✅ 1500 palavras (conforme planejado)
- ✅ SEO otimizado (keyword primária presente)
- ✅ Brand voice consistente
- ✅ CTA claro no final
- ✅ Sem erros de ortografia

**Observações:**
- Artigo ficou excelente, storytelling envolvente
- Sugestão: adicionar mais 1 link interno para case relacionado

**Próxima tarefa:** Tarefa 2.2 (Cover image)
```

### Para Fase Completa

```markdown
## Revisão: Fase 2 — Conteúdo

**Status:** ✅ Fase aprovada

**Tarefas:**
- ✅ Tarefa 2.1: Artigo de blog
- ✅ Tarefa 2.2: Cover image
- ✅ Tarefa 2.3: Posts LinkedIn

**Consistência:**
- ✅ Brand voice uniforme em todas as peças
- ✅ Mensagens-chave aparecem consistentemente
- ✅ Tom profissional mas acessível mantido

**Issues:**
- Nenhum issue crítico
- Issue menor: Post 3 do LinkedIn poderia ter CTA mais forte

**Recomendação:**
- Corrigir CTA do Post 3 na próxima iteração
- Prosseguir para Fase 3
```

### Para Projeto Completo

```markdown
## Revisão Final: Lançamento Produto X

**Status:** ✅ Projeto aprovado

**Entregas:** 10/10 completas
- 1 briefing completo
- 1 identidade visual
- 2 personas detalhadas
- 1 artigo de blog (1500 palavras)
- 1 cover image

- 1 whitepaper (3000 palavras)

**Qualidade:**
- ✅ Brand voice consistente em todas as peças
- ✅ SEO otimizado (score 85/100)
- ✅ Acessibilidade (alt text em todas as imagens)
- ✅ CTAs claros e adequados ao funil
- ✅ Formatação profissional

**Alinhamento com Briefing:**
- ✅ Objetivo: gerar 50 leads qualificados

1. Aprovar para publicação
2. Agendar posts conforme calendário
3. Monitorar métricas após 30 dias
4. Iterar baseado em performance

**Próximo passo:** Finalização com usuário
```

## Red Flags

**Nunca:**
- Pule revisão (mesmo para tarefas "simples")
- Aceite entregável sem validar critério de sucesso
- Ignore issues de brand voice
- Aprovar projeto com issues críticos
- Esqueça de documentar revisão

**Sempre:**
- Revise cada entrega individualmente
- Valide consistência entre peças
- Confirme alinhamento com briefing
- Documente issues e recomendações
- Forneça feedback construtivo
